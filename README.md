# 2人の記念日カウント

「記念日を保存するだけ」で終わらず、**忘れない・続けたくなる習慣化**をテーマにした、スマホ最優先の軽量Webアプリです。  
HTML / CSS / Vanilla JavaScript のみで構成し、外部ライブラリなしで GitHub Pages にそのまま配置できます。

## アプリ概要

このアプリでは、2人の名前・付き合った日・大切な記念日を保存しながら、毎日開きたくなる仕組みも同時に提供します。

- 記念日の登録と一覧表示
- 7日前 / 3日前 / 前日 / 当日の疑似リマインド
- 日替わりで変わる「今日のひとこと」
- 連続利用日数（ストリーク）と今月の利用回数の可視化
- よくある記念日のクイック追加
- スマホ向け下部固定ナビゲーション
- 初回ガイド付きの空状態表示
- localStorage の整合性チェックと安全な初期化
- テーマカラー切り替えと通知ON/OFF設定

## 習慣化機能

### 1. 疑似リマインド機能

プッシュ通知は使わず、ブラウザ画面内に通知カードを表示します。

通知条件:

- 7日前
- 3日前
- 前日
- 当日

表示例:

- `初デートまであと3日です。`
- `記念日ディナーは今日です。`

### 2. 今日のひとこと

日付ベースで毎日メッセージが切り替わります。

- 出会って何日目か
- 次の記念日まであと何日か
- 写真を見返す・予定を考えるなどの軽い行動提案
- 12パターンの固定メッセージを用意

### 3. 継続利用の見える化

`localStorage` に開いた日を保存し、次を表示します。

- 連続利用日数（ストリーク）
- 今月の利用回数

これにより、「記録する」だけでなく「毎日少しだけ開く」行動を促します。

### 4. クイック追加

以下の定番イベントをワンタップでフォーム入力できます。

- 初デート
- 初旅行
- 告白
- 記念日ディナー

## 通知の仕組み

このアプリの通知は**ブラウザ内のみ**です。

- Web Push は使用していません
- ページを開いたときに、記念日との日数差を計算して通知UIを表示します
- 設定画面の「通知表示」をOFFにすると、通知エリアは非表示になります

そのため、GitHub Pages のような静的ホスティングでもそのまま動作します。

## GitHub Pages 公開手順

1. このリポジトリを GitHub に push します。
2. GitHub の対象リポジトリで **Settings** を開きます。
3. 左メニューの **Pages** を選択します。
4. **Build and deployment** で `Deploy from a branch` を選びます。
5. 公開したいブランチを選びます。
6. フォルダに `/ (root)` を指定します。
7. 保存後、数分待つと公開URLが発行されます。
8. 発行されたURLで `index.html` がそのまま表示されます。

## localStorage 仕様

保存キー:

- `anniversary-counter-habit-v1`

保存構造:

- `settings`
  - `personOne`
  - `personTwo`
  - `relationshipDate`
  - `notificationsEnabled`
  - `themeColor`
- `data`
  - `anniversaries[]`
- `stats`
  - `lastOpenedDate`
  - `streak`
  - `openedDates[]`

### 整合性チェック

起動時に次を検証します。

- JSONとして読めるか
- `settings / data / stats` の形が壊れていないか
- 日付文字列が `YYYY-MM-DD` 形式か
- 配列や数値が不正なら安全な初期値に戻せるか

不正データがあっても、アプリ全体が止まらないよう `try-catch` と正規化処理で保護しています。

## データ消失の注意点

このアプリはサーバー保存を行わず、ブラウザの `localStorage` にだけ保存します。

そのため以下のケースではデータが消える可能性があります。

- ブラウザの閲覧データ削除
- 端末変更
- シークレットモード利用
- ブラウザやOSの自動ストレージ削除

大切なデータを長期保存したい場合は、将来的にエクスポート機能やPWA対応を追加する設計へ拡張しやすい構成にしています。

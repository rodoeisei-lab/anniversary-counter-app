# ふたりカウント（記念日カウントWebアプリ）

## アプリ概要
「ただ日数を数える」だけではなく、**今日の2人にとって意味のある情報をすぐ見られる**ことを重視した、スマホ最優先のWebアプリです。  
GitHub Pagesでそのまま動くよう、HTML / CSS / Vanilla JavaScriptのみで構成しています。

## 改善コンセプト
- 見るだけでも価値があるホーム
- 押しつけない、やさしい提案
- 入力は最小限、振り返りは気持ちよく
- 将来の通知・PWA化を見据えた保存設計

## 主な機能
- **ホーム（今日の2人）**
  - 今日の日付
  - 付き合ってからの累計日数
  - 今月の月記念日まで
  - 次の年記念日まで
  - 今日のひとこと（条件＋日付シードで変化）
  - 今日のおすすめアクション（1〜3件）
  - 最近近い予定1件
  - 最近追加した記念日1件
  - 思い出メモ（最新またはお気に入り）
- **一覧（3タブ）**
  - 記念日 / 予定 / 思い出メモ
  - 検索
  - 並び替え
  - 種別フィルタ
  - 過去 / 未来 / すべて
  - お気に入りのみ
  - タグ絞り込み（思い出メモ）
  - コンパクト表示 + 詳細展開
- **振り返り**
  - 最近の思い出メモ
  - お気に入りメモ
  - 過去の記念日
  - 過去の予定
  - 今月のふりかえり件数
  - 最後に記録した日
- **追加**
  - すぐ追加（思い出メモ/予定/テンプレート）
  - しっかり追加（記念日 / 予定 / 詳細付き思い出メモ）
- **バックアップ / 復元**
  - 全データJSONエクスポート
  - JSONインポート
  - スキーマ簡易チェック
  - 復元前確認
  - 成功/失敗メッセージ
  - サンプルデータ読込

## 画面の役割
- **ホーム**: 今日の確認（開いた瞬間に必要な情報）
- **一覧**: データ管理（検索・絞り込み・削除）
- **振り返り**: 見返す楽しさ（蓄積実感）
- **追加**: 入力導線（すぐ追加 / しっかり追加）

## データの違い
- **記念日**: 節目や過去イベント
- **予定**: これからの行動予定
- **思い出メモ**: 日常の短い感情や会話（タグ・お気に入り付き）

## データ保存仕様
localStorageで以下を分離保存し、壊れたデータ時はフォールバックします。

### localStorageキー構成
- `anniv.settings.v1`
- `anniv.anniversaries.v1`
- `anniv.plans.v1`
- `anniv.memories.v1`
- `anniv.uiState.v1`

### 各レコード共通
- `id`
- `createdAt`
- `updatedAt`

### 例
```json
{
  "id": "mem_xxxxx",
  "body": "帰り道の会話がよかった",
  "title": "帰り道の会話",
  "date": "2026-03-25",
  "tag": "会話",
  "relatedAnniversaryId": "",
  "favorite": true,
  "createdAt": "2026-03-25T10:00:00.000Z",
  "updatedAt": "2026-03-25T10:00:00.000Z"
}
```

## JSONバックアップ / 復元方法
1. 追加画面の「バックアップ / 復元」を開く
2. `JSONを書き出す`でエクスポート
3. `JSONを読み込む`でインポート
4. 復元前確認ダイアログで承認
5. 成功時は通知、失敗時はエラー文を表示

## ローカル起動方法
### 1) 直接開く
`index.html` をブラウザで開くだけで動作します。

### 2) ローカルサーバー
```bash
python3 -m http.server 8000
```
`http://localhost:8000` を開いてください。

## GitHub Pages公開方法
1. GitHubへpush
2. **Settings > Pages**
3. **Deploy from a branch**
4. ブランチ（例: main）+ `/(root)` を選択
5. 公開URLにアクセス
